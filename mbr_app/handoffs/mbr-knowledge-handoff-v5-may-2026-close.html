6. Methodology Locks Ratified This Session

Customer Cohort Definitions

  • Weekly / Bi-Weekly: ≥24 unique purchase weeks/year (combines old Weekly 48+ and Bi-Weekly 24–47)
  • Monthly: 12–23 unique purchase weeks/year
  • Casual: 2–11 unique purchase weeks/year 2-week floor, not 1
  • New To Walmart: 0 or 1 purchases

Cohort Snapshot Table — Locked Column Set

Cohort | SOV | Impressions | CPTS | CTR | ATC/1K | GMV ($M) | ExpDel | GMF | OG

  • Use .kpi table class
  • Highlight winning metrics with .up (green bold)
  • Append Blended row using .grand-total styling
  • Always include Casual + NTW even when narrative focuses on Monthly + Weekly/Bi-Weekly

Two-Engine Visual — Implementation Pattern

  • Parent: display:grid; grid-template-columns:1fr 1fr; gap:20px; align-items:stretch
  • Each card: display:flex; flex-direction:column; border:2px solid <theme>; border-radius:10px
  • Stat row: 3-column CSS grid with tabular-nums for big numbers
  • Body: flex column with flex:1 so pills bottom-anchor via margin-top:auto
  • Persona paragraph: min-height:4.5em keeps STRATEGY headers cross-aligned
  • Color themes: Monthly = #0053e2 blue + #eff6ff bg | Weekly/Bi-Weekly = #b8860b gold + #fffbeb bg

Section Insight Labels — Locked

  • 'Callout:' — preferred for mid-section insight callouts inline within body paragraphs
  • 'Takeaway:' — acceptable for section-opener summary insights (1-line)
  • AVOID 'Read:' — deprecated May 2026 (too instructional)

Owner-Team Accountability — Locked

  • Embed explicit owner-team labels in strategic recommendations + visuals
  • Color-coded pills matching section/cohort visual
  • Standard recurring owners: Home Page Features, Site Optimization, Site Content, Customer Strategy, WMC, W+ Team, Hero Carousel Product, Merch Ops, Marketing/SEO, Instrumentation Eng
  • Pill pattern: <div style="background:<color>; color:white; padding:10px 14px; border-radius:6px; font-size:13px; font-weight:600; display:inline-block;">🏗️ Owned by: <Team></div>

  2. Renumber section's own h2 + sub-headings
  3. Insert at new position
  4. Renumber ALL downstream h2 sections AND h3 sub-headings
  5. Grep for cross-references using old section numbers; update
  6. Update TOC if present
  7. Verify with grep: zero occurrences of old section numbers in h2/h3 contexts

8. Process Lessons Learned

  1. Plan for 5–7 director-edit cycles per MBR section refresh — May 2026 had 5 version bumps post-initial (v11 → v16). Common cadence: add new content → reframe → reposition → polish labels/alignment → final lock.
  2. File-based publish for share-puppy is non-negotiable — write to disk, verify with grep, then upload. Paste-through still introduces typos in long blocks.
  3. Single-visual > multi-table-and-list combos when communicating strategic dichotomies. Directors consistently prefer one strong visual to three smaller assets.
  4. Owner-team accountability transforms strategy from analysis to action — adding the owner pill makes recommendations actionable rather than observational.
  5. Cohort definitions belong in Appendix, not in section narrative — keeps deep dive sections strategic, not definitional.
  6. CSS Grid + flex stretch is the answer to alignment — flexbox wrap leaves cards unequal height; grid with align-items:stretch + margin-top:auto on pinned bottom elements is the locked pattern.
  7. Casual cohort starts at 2 weeks, not 1 — the 1-week floor would overlap NTW's "0 or 1 purchases" definition. Always cross-check cohort boundaries for non-overlapping coverage.

12. SQL Patterns Added This Cycle

Q-COHORT-SNAPSHOT — Customer Cohort Performance

SELECT
  customer_cohort,
  SUM(module_view_count) AS impressions,
  SAFE_DIVIDE(SUM(overall_click_count), SUM(hp_session_count)) * 1000 AS cpts,
  SAFE_DIVIDE(SUM(overall_click_count), SUM(module_view_count)) * 100 AS ctr_pct,
  SAFE_DIVIDE(SUM(total_atc_count), SUM(module_view_count)) * 1000 AS atc_per_1k,
  SUM(total_gmv) AS gmv,
  SUM(exp_del_activation_flag) AS exp_del,
  SUM(gmf_activation_flag) AS gmf,
  SUM(og_activation_flag) AS og
FROM `wmt-site-content-strategy.scs_production.hp_summary_asset`
WHERE session_start_dt BETWEEN '<start>' AND '<end>'
  AND experience_lvl2 IN ('App: iOS','App: Android','Web: Desktop','Web: Mobile')
  AND IFNULL(moduletype,'') NOT IN ('InteractiveImageCarousel','socialCarouselV2')
GROUP BY customer_cohort;

Q-COHORT-MESSAGE — Message × Cohort cross-tab (for diagnostic Q in §9.1)

Group by (customer_cohort, message_name) for top 20 messages by blended impressions. Returns which messages over-index for each cohort.

Q-COHORT-MOM — Cohort drift MoM (June MBR+)

Same shape as Q-COHORT-SNAPSHOT but with month-bucket case statement. Returns May vs June SOV, CTR, ATC/1K, activation share by cohort.
