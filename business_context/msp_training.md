## 🧭 What Is MSP?

**Message Strategy Planning (MSP)** is a holistic site planning process that:
- Creates a **source of truth** for ALL messages on site
- Enables front-end message activation on Walmart.com
- Passes messages to P13N (personalization) to scale via algo-driven decision engines
- Tracks message status across all business units
- Powers unique message identifiers for data capture and reporting

---

## 📌 What Is a Message?

> *"A message is any idea we want to communicate to our customer."*

Previously called: story, theme, campaign, mission, moment.

Messages can be:
- **Single use** — one-time campaign
- **Repeated use** — occurs annually (e.g., Easter, Back to School)
- **Always on** — always eligible (e.g., "Continue Your Shopping")

## 🌲 Message Hierarchy (M0 → M4)

Messages are organized in a hierarchical structure:

```
M0 — Primary Message
  │
  ├── M1 — Sub-message 1
  ├── M1 — Sub-message 2
  │     │
  │     ├── M2 — Sub-message of M1
  │     │     │
  │     │     ├── M3 — Sub-message of M2
  │     │     │     │
  │     │     │     └── M4 — Deepest level
```

**Example: Halloween**
```
M0: Halloween
  ├── M1: Halloween Costumes
  │     ├── M2: Halloween Costumes for Adults
  │     ├── M2: Halloween Costumes for Kids
  │     │     └── M3: Halloween Costumes for Dogs
  │     │           └── M4: Halloween Costumes for Cat
  │     └── M2: Halloween Costumes for Pets
  ├── M1: Halloween Decor
  └── M1: Halloween Candy
```

> ⚠️ **M0 does NOT indicate tier, rank, or priority** — it is just the top of the hierarchy.

## 🛠️ MMUI — Message Management UI

The **MMUI** (CX Tempo) is the tool used to run the MSP process:
- Integrated with CX Tempo
- The **single source of truth** for all approved messages + their metadata
- Systemically passes messages to P13N for algo-driven placement
- Unique message identifiers enable data capture and reporting

### Message Status Flow in MMUI

| Status | Definition |
|--------|-----------|
| **DRAFT** | Approved message added to MMUI by SCS team; Message Owner assigned |
| **IN PROGRESS** | Message Owner has completed at least one required field |
| **READY FOR REVIEW** | All required fields complete AND an asset is available for review |
| **READY FOR GO LIVE** | Marked "Ready for Go Live"; awaiting start date |
| **LIVE** | Ready for Live AND start/end dates are in range |
| **EXPIRED** | Message end date has passed (within last 7 days) |

### Dashboard Color Coding (Activation Queue)
- 🔴 **Red** — Less than 1 week till live: **requires immediate action**
- 🟠 **Orange** — Less than 2 weeks till live: requires action to finalize
- 🟡 **Yellow** — More than 2 weeks till live: upcoming action needed
- ✅ **Goal**: Get messages to GREEN (Ready to Go Live) status before 1 week out

## 📝 Message Submission Process

### When to Submit via MSP
All 4 must be YES:
1. Is this an idea I want to overtly message to the customer?
2. Does it clearly convey what we're trying to help the customer do (solve a problem, inspire, showcase a benefit)?
3. Do we have an assortment of products or clear call to action?
4. Will the message include more than navigational assets?

> ❌ If NO to any: Not an MSP message. Consider: item pinning, SEO optimization, search redirects, navigation.

### Required Submission Fields
| Field | Description |
|-------|-------------|
| Submission Date | Date of submission |
| Message Requestor | Full name |
| Requested Message Name | Clear, concise, customer-facing; limit special characters |
| Description | What are we helping customer do? What products? Who resonates? |
| Site Start & End Dates | MM/DD/YYYY format |
| Message Sponsor | Business unit |
| Department | Department |
| Message Owner | Individual executing/building the message |
| Editorial Owner | Editorial partner for asset kick-off |
| Linking Destination | Where message drives to |

### Submission Response SLA
- **Urgent**: 3 business days
- **Standard**: Up to 10 business days
- *M0 Message Frameworks may take longer due to additional reviews*

## 🎯 Personas

- **Purpose**: Tell the P13N algo who is most relevant for a message — 'soft' targeting, not hard targeting
- **Rule**: Minimum 1, maximum 5 personas per message
- **How it works**: Kick starter that tells the algo who to serve content to; algo then layers in individual customer signals
- **Optional targeting**: Geo-location, W+ cohort, and other targeting fields available

## 📍 ATF Homepage Eligibility Rules

### General HP Eligibility (Message Owner responsible):
1. Homepage listed in **Page Eligibility in MMUI**
2. Active asset in **needed size** available in MMUI
3. Message in **Ready to Go** status
4. Start and End Date within time frame

### ATF (Above the Fold) Eligibility Guardrails:
```
IF time between site_start and event_start ≤ 7 days:
    THEN message is ATF eligible 3 days before event_start
ELSE:
    THEN message is ATF eligible when (current_date to event_date) ≤ 21 days
```

> ⚠️ If message has NO event date in MMUI → message is **NEVER** ATF eligible
> ⚠️ Message types that are NOT ATF eligible without override: brands, evergreen, CSR, Flash Deals

### Pages Required for HP Eligibility:
- `Web – Glass HomePageDesktopV1`
- `Web – Glass HomePageMwebV1`
- `App – MobileHomeScreen`

## 🔄 HP Algorithm Override — When Is It Approved?

Algorithm overrides (static or rotation ATF placement) are considered for:

| Use Case | Examples |
|----------|---------|
| Deal events + select enterprise-wide campaigns | Deals for Days, Black Friday, Thanksgiving |
| Geo-targeted campaigns based on customer need | Winter Weather Prep, Allergy Essentials |
| High-profile, in-demand national brands | Apple launches, Bettergoods |
| New business lines | Walmart+, Walmart Connect modules |
| Dedicated CSR modules | Veteran-owned business, DE&I campaigns |

### HP Algorithm Override Process:
1. Message Owner fills out ALL HP Algorithm Override columns in submission form
2. SCS vets the request and presents at **Biweekly Leadership Vetting Meeting (LVM)**
3. SCS updates monthly calendar with approved requests

## 🏗️ Navigation vs. Messages

| | Message | Navigation |
|-|---------|-----------|
| **What it is** | Idea communicated to customer to inspire/help | Drives customer deeper into site funnel |
| **Examples** | Valentine's Day, Back to School, Bettergoods | Category hubspokes (Pants, Jeans, Belts) |
| **Asset type** | Adjustable Banner, Triple Pack, TTO, etc. | Hubspokes (text/logo/lifestyle), merch chips |
| **MSP submission** | Required | NOT required for navigation-only |
| **Requested via** | MSP Message Submission Form | Workfront or Smart Creative Self-Service |

> 💡 Exception: Messages with BOTH navigation AND additional assets (e.g., Bettergoods with TTO + Skinny Banner + Adj Banner) → still require MSP submission.
